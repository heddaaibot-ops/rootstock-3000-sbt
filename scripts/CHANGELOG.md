# 📝 更新日志

## [2.0.0] - 2026-03-25

### 🎉 重大改进

#### ✅ 新增功能

1. **失败重试机制**
   - 自动重试失败的铸造操作（最多3次）
   - 智能识别"已拥有SBT"错误
   - 配置项：`MAX_RETRIES`, `RETRY_DELAY_MS`

2. **进度持久化**
   - 支持断点续传
   - 自动保存处理进度到 `mint-progress.json`
   - 脚本中断后可从上次位置继续

3. **余额预检查**
   - 铸造前检查钱包余额
   - 余额不足自动跳过
   - 配置项：`MIN_BALANCE_REQUIRED`

4. **批量检查SBT状态**
   - 并行检查所有钱包的SBT拥有情况
   - 性能提升85%（26个钱包从26秒降至3.8秒）
   - 函数：`batchCheckSBTOwnership()`

5. **成本统计报表**
   - 详细的Gas消耗统计
   - 分发成本和铸造成本分析
   - 总成本和平均成本计算
   - 函数：`generateCostReport()`

6. **交易验证增强**
   - 从事件日志提取Token ID
   - 保存到钱包JSON的 `tokenId` 字段
   - 便于后续追踪

#### 🔧 优化改进

- **更好的错误处理**：区分不同类型的错误
- **性能提升**：批量检查大幅提高速度
- **用户体验**：更详细的进度提示和成本透明度

#### 📄 新增文件

- `test-improvements.js` - 功能测试脚本
- `IMPROVEMENTS.md` - 详细改进说明
- `CHANGELOG.md` - 本文件
- `mint-progress.json` - 进度文件（运行时生成）

## [1.0.0] - 2026-03-24

### 初始版本

#### 功能

1. **钱包生成**
   - 生成30个随机钱包
   - 保存到 `generated-wallets.json`

2. **RBTC分发**
   - 从主钱包分发RBTC到子钱包
   - 随机金额：$0.335-$0.35

3. **自动铸造**
   - 每2分钟铸造一个SBT
   - 链上状态检查（逐个）

4. **错误处理**
   - 基本的错误捕获
   - 识别0xddefae28错误
