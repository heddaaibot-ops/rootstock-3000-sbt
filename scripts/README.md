# Rootstock 3000 SBT 自动铸造脚本

这个脚本用于自动化生成钱包、分发 rBTC 并铸造 SBT。

## 🚀 快速开始

### 1. 安装依赖

```bash
cd scripts
npm install
```

### 2. 配置 RBTC 价格

编辑 `auto-mint.js` 文件，更新当前 RBTC 价格：

```javascript
const CONFIG = {
  // ...
  RBTC_PRICE_USD: 100000, // 根据实际价格更新
  // ...
};
```

### 3. 使用方法

#### 方式一：分步执行（推荐）

```bash
# 步骤 1: 生成 30 个钱包
node auto-mint.js generate

# 步骤 2: 从主钱包分发 rBTC 到 28 个钱包
node auto-mint.js distribute <主钱包私钥>

# 步骤 3: 自动铸造 SBT（每 2 分钟一个）
node auto-mint.js mint
```

#### 方式二：一键执行

```bash
# 执行所有步骤
node auto-mint.js all <主钱包私钥>
```

## 📋 脚本功能

### 步骤 1: 生成钱包

- 生成 30 个新的随机钱包
- 保存钱包信息到 `generated-wallets.json`
- 包含地址、私钥、助记词

### 步骤 2: 分发 rBTC

- 从主钱包向 28 个钱包分发 rBTC
- 每个钱包获得随机金额（0.335-0.35 USD 等值的 rBTC）
- 自动检查主钱包余额是否足够
- 逐个发送交易，避免 nonce 冲突

### 步骤 3: 自动铸造

- 使用 28 个钱包依次铸造 SBT
- 每次铸造间隔 2 分钟
- 自动记录铸造状态和交易哈希
- 出错时自动跳过并继续

## 📊 配置选项

在 `auto-mint.js` 的 `CONFIG` 对象中可以修改：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `RBTC_PRICE_USD` | RBTC 价格（USD） | 100000 |
| `MIN_AMOUNT_USD` | 最小分发金额（USD） | 0.335 |
| `MAX_AMOUNT_USD` | 最大分发金额（USD） | 0.35 |
| `MINT_INTERVAL_MS` | 铸造间隔（毫秒） | 120000 (2分钟) |
| `TOTAL_WALLETS` | 生成钱包总数 | 30 |
| `WALLETS_TO_FUND` | 资助钱包数量 | 28 |
| `GAS_LIMIT` | Gas 限制 | 150000 |

包含所有生成的钱包信息：

```json

    "address": "0x...",
    "privateKey": "0x...",
    "mnemonic": "word word word...",
    "funded": true,
    "fundedAmount": 0.00000345,
    "minted": true,
    "mintTxHash": "0x...",
    "mintTime": "2026-03-24T14:30:00.000Z"
  }

```

⚠️ **重要**: 这个文件包含私钥，请妥善保管，不要泄露！

## 💰 主钱包余额要求

假设 RBTC = $100,000：
- 每个钱包约需：0.0000035 RBTC ($0.35)
- 28 个钱包约需：0.000098 RBTC ($9.8)
- Gas 费用约：0.000042 RBTC ($4.2)
- **总需求约：0.00014 RBTC ($14)**

建议准备 **0.0002 RBTC ($20)** 以确保充足。

## 📊 执行时间估算

- 生成钱包：< 5 秒
- 分发 rBTC：约 1 分钟（28 笔交易）
- 铸造 SBT：约 56 分钟（28 次铸造 × 2 分钟间隔）

**总时间约：1 小时**

## 🛠️ 故障排除

### 问题：分发时余额不足

**解决**: 确保主钱包有足够的 rBTC（建议 0.0002 RBTC）

### 问题：铸造失败 "Already minted"

**解决**: 脚本会自动跳过已铸造的钱包，继续下一个

### 问题：交易卡住

**解决**:
1. 检查 RPC 节点是否正常
2. 尝试更换 RPC 地址（编辑 `CONFIG.RPC_URL`）

### 问题：想要暂停/恢复

- 脚本会保存进度到 `generated-wallets.json`
- 可以随时 Ctrl+C 暂停
- 重新运行相应步骤会自动跳过已完成的钱包

## 🔒 安全建议

1. ✅ 不要将 `generated-wallets.json` 上传到 Git
2. ✅ 使用完后备份并删除敏感文件
3. ✅ 主钱包私钥通过命令行参数传入（不保存在代码中）
4. ✅ 脚本执行完后，可以将资金转回主钱包
